嵌入式学习笔记（一）：16届蓝桥杯省赛真题学习

首先我们需要知道真题题目
https://www.lanqiao.cn/courses/2786/learning/?id=4220710&compatibility=false
这是蓝桥杯真题官网，在里面找到16届嵌入式省赛，我们先看硬件配置

我们可以知道，整体的功能实现由G431作为主控芯片，接收两路模拟信号输入，PA15作为脉冲信号输入捕获，PA7输出可控PWM波，
还有包括LCD，LED灯和按键。
知道了总体结构和模块，那么我们就要对每个模块进行一个引脚分配，由官方提供的开发板硬件原理图我们可以发现，板子上挂载的8个
LED灯是由PC8PC15来控制，4个按键是由PB0PB2和PA0来控制，模拟信号由电位器触发，题目中要求我们使用R37，R38作为两路
模拟信号触发源，然后PA15作为脉冲信号输入捕获，PA7输出可控PWM波，LCD部分由官方提供的程序文件自动配置。
引脚分配好之后，就可以去到我们的CubeMX里面给每个部分进行一个初始化配置，如图

配置好后，直接生成代码，软件会根据CubeMX里的配置自动生成初始化函数。
代码生成好后，我们先来编写最基础的LED和按键控制代码，首先是LED控制，因为官方板子的LED口和LCD口是接通的，所以每次改变
LED时我们需要通过PD2的锁存器来控制，不然当你改变LED的时候可能会导致LCD出现异常。

//led灯控制
void Led_Disp(uint8_t dsLed)
{
HAL_GPIO_WritePin(GPIOC,GPIO_PIN_All,GPIO_PIN_SET);//每次调用将PC口全部置为高电平，LED灯全灭
HAL_GPIO_WritePin(GPIOC,dsLed<<8,GPIO_PIN_RESET);//在PC的16位IO口中，板子上的灯挂载在高八位，采用左移八位来控制
HAL_GPIO_WritePin(GPIOD,GPIO_PIN_2,GPIO_PIN_SET);//PD2为锁存器，每次控制LED后应该打开锁存器，让控制生效
HAL_GPIO_WritePin(GPIOD,GPIO_PIN_2,GPIO_PIN_RESET);//生效完毕后关闭锁存器
}

这样，我们通过调用Led_Disp函数就可以实现对LED灯的控制，函数括号内填入16进制数，通过低八位来控制，每一位代表一个LED灯。
然后是按键扫描函数，我们通过状态机的形式实现，告别使用HAL_Delay这种阻塞性的语句，避免CPU卡死的情况出现。
接下来就是按键扫描函数的构建，按键扫描函数采用状态机的思想，用一个变量Key_State来表示目前状态，

//键盘扫描状态机
uint8_t Key_Scan(void)
{
static uint8_t Key_State=0;//状态0，等待按键按下
uint8_t Key_Return=0;//按键返回值，返回真实的案件编号
uint8_t Key_Press=0;//用于判断是否有按键按下，临时判断按键编号

if(HAL_GPIO_ReadPin(GPIOB,GPIO_PIN_0)==0)
Key_Press=1;
else if(HAL_GPIO_ReadPin(GPIOB,GPIO_PIN_1)==0)
Key_Press=2;
else if(HAL_GPIO_ReadPin(GPIOB,GPIO_PIN_2)==0)
Key_Press=3;
else if(HAL_GPIO_ReadPin(GPIOA,GPIO_PIN_0)==0)
Key_Press=4;

switch(Key_State)
{
case 0:
if(Key_Press!=0)//有键按下，准备进入状态1进行消抖
Key_State=1;
break;
case 1:
if(Key_Press!=0)//依然按下，表示不是抖动，按键编号赋值给Key_Return
{
Key_Return=Key_Press;
Key_State=2;
}
else
Key_State=0;
break;
case 2:
if(Key_Press==0)//状态2，等待按键松开，进行下一轮按键判断
Key_State=0;
break;
}
return Key_Return;
}

接下来我们要来实现PWM模块，按题中给的硬件图来看，我们需要用PA7来输出一个PWM波，这个PWM波的占空比和
频率与ADC的两路模拟信号R37和R38有关，根据题目信息，由占空比调节策略可得

共有n+1个台阶，步长DS=(DR-10)/N,那么N=(DR-10)/DS,台阶宽度Va=3.3/(N+1),那么我们可以得到此刻上了多少
台阶Current_step=(int)((a_vol)/3.3),用int实现向下取整，此时的占空比就为CD=DSCurrent_step+10，占空比最低
是10且为固定值。
然后回到我们的CubeMX，PA7选择TIM3_CH2作为PWM输出，预分频设置为80-1，自动重装载寄存器设置为1000-1，
生成代码到keil，编写PWM函数，
void PWM(void)
{
uint8_t N=(DR-10)/DS;
double Va=3.3/(N+1);
uint8_t Current_Step=(int)(a_vol/Va);
CD=DSCurrent_Step+10;

__HAL_TIM_SetCompare(&htim3,TIM_CHANNEL_2, CD * 10);
}
在主函数while中调用，烧录可以看到当我们拧动R37旋钮，占空比CD会有相应变化，在参数界面修改步长DS和最大值
DR，再拧R37，占空比变化步长和最大值也发生改变，功能成功实现。
到这里，我有一个疑问，如果我和其他任务一样，每50ms或者100ms调用一次PWM函数，那么产生出的PWM会断续吗，
事实上并不会，因为在你没有调用的49ms内，TIM会自动每1ms输出一次PWM，所以完全可以和其他任务一样，让他每
50ms执行一次，
//生成PWM
static uint32_t PWM_tick=0;
if(HAL_GetTick()-PWM_tick>=50)
{
PWM_tick+=50;
PWM();
}

同时在这里我发现了一个问题，就是当我的R37旋钮拧到最大值时，我的CD只能到达79，经查阅发现，原来是ADC
每次使用前都需要对其进行一次校准，校准由硬件自动完成，我们只需要在程序中调用校准程序，
HAL_ADCEx_Calibration_Start(&hadc1, ADC_SINGLE_ENDED);
HAL_ADCEx_Calibration_Start(&hadc2, ADC_SINGLE_ENDED);
同时我们也加上越界保护，防止拧到地超过最大值，将这个程序加在PWM函数中
if(Current_Step>N)Current_Step=N;

这个防越界程序可以配合引入工程裕量，就是把我们的最高电压3.3当成一个小一点的数比如3.1，这样做可以让我们
避免因为物理因素而使得电位器无法输出3.3V的最大值而出现的问题。
然后频率的调节策略和占空比类似，将程序写在计算占空比的程序后面即可，

uint8_t n=(FR-1000)/FS;
float Vb=3.3/(n+1);
uint8_t step=(int)(b_vol/Vb);
if(step>n)step=n;
CF=1000+step*FS;

uint32_t target_arr = (1000000 / CF) - 1;
uint32_t target_ccr = (target_arr + 1) * CD / 100;

__HAL_TIM_SET_AUTORELOAD(&htim3, target_arr); 
__HAL_TIM_SetCompare(&htim3, TIM_CHANNEL_2, target_ccr);

这里关于计算arr和ccr可能有点难理解，我们可以换个角度，系统主频经过预分频器后进入TIM定时器的频率是1MHZ，
那我们可以列出1/(1/1MHZ)=(arr+1)/(1/CF),计数器每秒计数量是一定的，arr寄存器在1/CF的时间内计数了arr+1次，
于是可得arr= (1000000 / CF) - 1,之后我们根据CCR的值等于PWM周期占空比计数速度，那么可列出式子
CCR=（1/CF）(CD/100)1/(1/1MHZ)其中1/(1/1MHZ)=(arr+1)/(1/CF)，所以可等同于
CCR=（1/CF）(CD/100)(arr+1)/(1/CF)。

至此，PWM输出功能也成功实现。
在这里，我又发现了一个问题，我拧动R37旋钮后不动，我的CD一直在抖动，这是因为物理噪声的原因，那么怎么消除
这个噪声呢？我们在ADC的读取任务里面加入一阶滤波程序，实现软件滤波，原理是把“上一次的旧电压”和
“这一次的新电压”按比例混合一下，旧的占大头，新的占小头。

// ADC读取，50ms进行一次
if(HAL_GetTick() - ADC_tick >= 50)
{
ADC_tick += 50;

// 1. 先把测到的新数据存到一个临时变量里
float new_a = Get_ADC_Vol(&hadc2);
float new_b = Get_ADC_Vol(&hadc1);

        // 新数据只占 20% 权重，老数据占 80% 权重
        a_vol = (a_vol * 0.8f) + (new_a * 0.2f);
        b_vol = (b_vol * 0.8f) + (new_b * 0.2f);
	}

这样，我们的CD就不会再抖动了。
