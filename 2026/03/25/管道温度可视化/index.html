管道温度可视化项目(一)：C++与VTK的虚实坐标映射

🚀 项目背景与功能描述

目前工业界主要的三维管道建模技术有以下四种：

  • ① CAD建模： 通过软件（如 AutoCAD Plant 3D、SolidWorks）完成几何建模。
  • ② BIM建模： 不仅包含几何形态，还涵盖材料、尺寸、维护等物理与功能属性，是形成“数字孪生”的基础。
  • ③ 图形图像建模： 利用采集到的纹理、轮廓、色彩等信息逆向生成三维模型。
  • ④ 点云建模： 通过三维扫描仪获取物体表面点云数据来完成建模。

管道三维建模不仅仅是几何图形的堆砌，更是物理管道系统与数字化模型的同步。通过传感器获取实时数据，可以在三维空间中进行监控、预测和诊断，极大提升运行效率。

📌 本项目的核心痛点与解决方案：
面对除尘房内复杂的管道布局和层叠现象，传统的数字表格难以直观反映设备的异常过热情况。为此，本项目通过构建管道的三维数字孪生模型，融合测温硬件数据，动态展示不同区域的连续温度场，实现直观的温度预警。

在此次方案中，我主要负责攻克两大核心功能：RFID测温标签温度映射 与 红外相机温度映射。
在实际应用中，RFID标签安置在管道上，巡检机器人携带读写器按规划路径读取温度，并传回上位机处理。检测模型如下图所示：

（一）管道温度映射：

实现管道温度可视化的第一步，是建立“数据”到“颜色”的映射桥梁。这一步我是在 Visual Studio 平台上，基于 C++ 和 VTK (Visualization Toolkit) 引擎来实现的。

在三维图形学中，一个 STL 管道模型是由成千上万个离散的顶点（Points）组成的。要让管道显示温度，核心思想就是：给模型的每一个顶点绑定一个温度值（标量），然后利用颜色查找表（Lookup Table）将温度值转换为肉眼可见的 RGB 颜色。

1. 为模型顶点绑定标量数据 (Scalars)

首先，我们需要读取 3D 模型，并创建一个浮点型数组，长度与模型的顶点数完全一致。系统初始化时，我将所有顶点的温度默认设置为 20.0℃（室温）。

2. 构建颜色映射表 (vtkLookupTable)

有了温度数据后，我们需要定一个规矩：多少度显示什么颜色？
这里我引入了 vtkLookupTable，将 20.0℃ ~ 100.0℃ 的温度区间切分成 256 个色阶。为了符合工业场景，低于 30℃ 的区域显示为冷色调（或底漆灰），超过 30℃ 的区域则使用线性插值计算对应的警告色（蓝 -> 红），从而起到强烈的视觉预警作用。

3. 贯通渲染管线 (Mapper & Actor)

最后，我们将带有温度数据的模型 (polydata) 和配置好的颜色表 (lut) 交给映射器 (Mapper) 处理，通过 Actor 投射到屏幕上。

为了验证颜色映射是否生效，我写了一段简单的测试逻辑：遍历所有顶点，以 X 轴坐标为自变量生成渐变温度。 离 X 轴原点越远，温度越高。

打通整条 VTK 渲染管线后，终于得到了管道温度的静态映射效果（如下图）。可以看到，温度场如预期般沿着 X 轴呈现出完美的冷暖渐变分布：

4. 工程化体验提升：交互式拾取器 (Picker)

为了更好的视觉效果，我在模型右侧添加了温度色带 (ScalarBar)，在左下角添加了 XYZ 三维坐标轴。

更重要的是，为了方便后期调试和运维人员巡检，我引入了 VTK 拾取器 (Picker)，实现了一套鼠标悬停探针功能。当鼠标指向具体管壁时，UI 会实时弹出该空间点位的 ID 编号与精准温度信息。
